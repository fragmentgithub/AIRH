# ひらがな知育アプリ 仕様書

## 1. 概要

4歳児向けに、ひらがな札を選んで身近なことばを完成させる知育ゲームアプリ。

絵や音をヒントに「ね」「こ」などのひらがなを順番に選び、「ねこ」のような短いことばを作る。正解すると、絵・音・スタンプなどで楽しく反応し、ひらがなの形・読み・ことばのまとまりに自然に触れられる体験を目指す。

## 2. コンセプト

### アプリ名 仮

「ひらがな ことばあそび」

### ねらい

- ひらがなを1文字ずつ覚えるだけでなく、文字を組み合わせるとことばになる感覚を育てる
- 4歳児が短時間で達成感を得られる
- 間違えても嫌にならない、やさしいゲーム体験にする
- 親が横で見守りやすく、遊びすぎを避けやすい設計にする

### 対象ユーザー

- メイン: 4歳の子ども
- サブ: 親、保護者

## 3. MVP 方針

最初に作る版では、機能を広げすぎず「1問ずつ、ひらがな札を選んでことばを完成させる」体験に集中する。

### MVP で作るもの

- トップ画面
- 問題画面
- 正解演出
- かんたんな進捗表示
- 2文字・3文字の単語データ
- 文字札の選択ロジック

### MVP では作らないもの

- ログイン
- 課金
- 広告
- オンラインランキング
- 複雑な保護者管理画面
- AIによる単語生成

## 4. 基本ゲームルール

1. 画面に単語のヒントを表示する
2. ヒントは絵、読み上げ、文字数の枠で構成する
3. 子どもは下に並んだひらがな札から、正しい文字を順番に選ぶ
4. 選んだ文字は上の枠に入る
5. すべて正しく入ると正解
6. 正解時は音・動き・スタンプなどで反応する
7. 次の問題に進む

### 例

- ヒント: ねこの絵
- 枠: `□ □`
- 選択肢: `ね` `こ` `あ` `ま`
- 正解: `ね` → `こ`
- 完成: `ねこ`

## 5. 学習設計

### 最初の難易度

4歳向けのため、初期は以下に限定する。

- 2文字の単語を中心にする
- 3文字は少なめにする
- 濁点、半濁点、小さい「ゃゅょっ」は初期では使わない
- 似た形の文字を同時に出しすぎない
- 1問あたりの選択肢は4つから開始する

### 難易度ステップ

| レベル | 内容 | 例 |
| --- | --- | --- |
| 1 | 2文字、選択肢4つ | ねこ、いぬ、もも |
| 2 | 2文字、似た音を少し混ぜる | はな、あめ、くま |
| 3 | 3文字、選択肢5つ | さかな、くるま |
| 4 | 3文字、順番の理解を強める | とまと、ばなな ※後半 |

### 初期単語リスト

| 単語 | 文字数 | カテゴリ | 備考 |
| --- | --- | --- | --- |
| ねこ | 2 | 動物 | 最初の問題候補 |
| いぬ | 2 | 動物 | 絵で伝わりやすい |
| くま | 2 | 動物 | 文字が見分けやすい |
| もも | 2 | 食べ物 | 同じ文字の反復 |
| あめ | 2 | 身近なもの | 絵と音で出しやすい |
| はな | 2 | 自然 | 視覚ヒント向き |
| うみ | 2 | 自然 | シンプル |
| そら | 2 | 自然 | シンプル |
| さかな | 3 | 動物 | 3文字入門 |
| くるま | 3 | 乗り物 | 3文字入門 |

## 6. 画面仕様

### 6.1 トップ画面

目的: 子どもがすぐ遊び始められること。

表示要素:

- アプリ名
- 大きな「はじめる」ボタン
- 今日のスタンプ数
- 小さな保護者向けリンク

操作:

- 「はじめる」を押すと問題画面へ進む

### 6.2 問題画面

目的: 絵や音をヒントに、ひらがな札を選ぶ。

表示要素:

- 問題番号
- ヒント画像
- 音をもう一度聞くボタン
- 文字を入れる枠
- ひらがな札
- やりなおしボタン

操作:

- ひらがな札をタップすると、次の空き枠に入る
- 入れた文字をタップすると取り消せる
- 音ボタンを押すと単語の読みを再生する
- 正解したら自動で正解演出へ進む

### 6.3 正解演出

目的: 達成感を出し、次もやりたくなる状態にする。

表示要素:

- 完成した単語
- ヒント画像の簡単なアニメーション
- 「できたね」などの短い肯定メッセージ
- 獲得スタンプ
- 「つぎへ」ボタン

操作:

- 「つぎへ」で次の問題へ進む
- 一定時間後に自動で次へ進んでもよい

### 6.4 保護者向け簡易画面 後回し

MVPでは必須ではない。後続版で追加する。

候補:

- 遊んだ問題数
- 正解数
- 苦手そうな文字
- 音のオンオフ
- 1回のプレイ問題数

## 7. UX ルール

### 子ども向け

- ボタンは大きくする
- 文字札は押しやすく、画面下部に置く
- 1画面に情報を詰め込みすぎない
- 正解・不正解の反応を強すぎないものにする
- 失敗時に「まちがい」と強く表示しない

### 間違えたとき

仕様:

- 間違った文字を選んでも即座に強い失敗演出を出さない
- 枠に入るが、完成時に違っていれば「もういっかい」とやさしく戻す
- 2回続けて迷った場合は、正しい文字札を少し光らせる

文言例:

- 「もういっかい」
- 「こっちかな？」
- 「よくみてみよう」

### 正解したとき

文言例:

- 「できたね」
- 「やったね」
- 「ねこ、できた」

演出例:

- スタンプが1つ増える
- ヒント画像が少し跳ねる
- 短い効果音を鳴らす

## 8. ビジュアル方針

### 雰囲気

- ゲームっぽい
- ただし派手すぎず、幼児が集中しやすい
- 絵本とミニゲームの中間

### デザイン

- 明るい背景
- 大きなひらがな
- 角丸の文字札
- 文字札は色分けしすぎず、読みやすさを優先
- キャラクターやスタンプ要素で楽しさを出す

### 避けること

- 広告っぽい見た目
- 小さいボタン
- 長い説明文
- 多すぎるメニュー
- 失敗を強調する赤いバツ演出

## 9. データ仕様

### 単語データ

```json
{
  "id": "neko",
  "word": "ねこ",
  "reading": "ねこ",
  "category": "animal",
  "level": 1,
  "imageKey": "cat",
  "choices": ["ね", "こ", "あ", "ま"]
}
```

### フィールド

| フィールド | 内容 |
| --- | --- |
| id | 単語ID |
| word | 正解のひらがな |
| reading | 読み上げ用テキスト |
| category | 動物、食べ物、自然など |
| level | 難易度 |
| imageKey | 表示画像のキー |
| choices | 表示する文字札 |

## 10. 状態管理

### 画面状態

- currentScreen: `home` / `quiz` / `success`
- currentQuestionIndex
- selectedLetters
- completedWords
- stampCount

### 判定

- `selectedLetters.join("") === currentQuestion.word` なら正解
- 文字数が足りない場合は判定しない
- 文字数がそろって不正解なら、軽いフィードバック後に入力を戻す

## 11. 音声仕様

MVPではブラウザの読み上げ機能または事前用意した短い音声を使う。

### 音の種類

- 単語読み上げ
- 文字読み上げ
- 正解音
- タップ音

### 注意

- 初期状態で音量が大きすぎないようにする
- ミュート切り替えを用意できるとよい
- 読み上げは短く、連続再生しすぎない

## 12. プレイ時間設計

4歳向けのため、1回のプレイは短くする。

- 1セット: 5問
- 1問: 15〜30秒程度
- 1セット完了後に「おしまい」画面を出す
- 連続プレイはできるが、区切りを明確にする

## 13. MVP の受け入れ条件

- トップ画面からゲームを開始できる
- 2文字単語を3問以上遊べる
- 文字札をタップして枠に入れられる
- 正しい順番で選ぶと正解になる
- 正解時に視覚的な反応がある
- 不正解時にやさしいやり直しができる
- スマホ幅でも文字札が押しやすい
- 4歳児が親の補助なしで基本操作できる

## 14. 初期実装タスク

1. 画面構成を作る
2. 単語データを定義する
3. 問題表示ロジックを作る

5. 正解判定を作る
6. 正解演出を作る
7. スマホ表示を調整する
